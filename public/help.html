סקירה כללית – מה זה IronBend?
מערכת ניהול מרכזית לבית מלאכה לכיפוף ברזל

IronBend היא מערכת ניהול מלאה לבית מלאכה לכיפוף ברזל לבניין. המערכת מנהלת את כל מחזור החיים של ההזמנה — מקבלת ההזמנה מהלקוח ועד לשליחה עם הנהג לאתר הבנייה.

הזמנות
קבלה, עריכה, ותיכנות הזמנות ברזל

יצירת הזמנה חדשה

  1. לחץ ➕ הזמנה חדשה בפינה הימנית העליונה
  2. הכנס שם לקוח / מספר אתר / כתובת
  3. הוסף פריטים — לכל פריט: צורה, קוטר ברזל, כמות, מידות
  4. בחר תאריך אספקה
  5. לחץ 💾 שמור

עריכת צורת ברזל

לחיצה על שורת פריט פותחת את עורך הצורות. ניתן לבחור מתוך ספריית צורות מוכנות (לפי תקן ישראלי) או לצייר צורה מותאמת אישית.

💡
תצוגה 3D / 2D לחץ על 🧊 3D לראיית הצורה בפרספקטיבה תלת-ממדית. לחץ 📐 2D לחזרה לתצוגה שטוחה עם מידות.

שליחת הזמנה למכונה

לאחר שמירה, לחץ 📡 שלח למכונה. הפרמטרים מועברים אוטומטית למכונת הכיפוף דרך Modbus. אם יש מספר מכונות — המערכת בוחרת את המכונה הפנויה המתאימה לפי טווח הקוטר.

🔧
מכונות Modbus
הגדרת חיבור ושליחת פרמטרים למכונות כיפוף
📡
מה זה Modbus? Modbus הוא פרוטוקול תקשורת תעשייתי. IronBend שולח פרמטרי כיפוף (זוויות, אורכים, כמות) ישירות למכונה דרך רשת או כבל סיריאלי — ללא הקלדה ידנית של האופרטור.

סוגי חיבור

סוגמתי משתמשיםציוד נדרש
🌐 TCP/IP מכונה מחוברת לרשת דרך ממיר Ethernet כמו USR-N510 כבל Ethernet, גיטוויי TCP→RS485
🔌 RTU סיריאלי השרת מחובר ישירות למכונה בכבל RS-485 / RS-232 כבל RS-485, מתאם USB→RS485

הוספת מכונה חדשה

  1. עבור לטאב ניהול → מכונות
  2. לחץ ➕ מכונה חדשה
  3. הכנס שם ותווית (אות/ספרה לזיהוי מהיר)
  4. בחר סוג חיבור: TCP/IP או RTU סיריאלי
  5. מלא את שדות הכתובת (IP:פורט לTCP, פורט COM לRTU)
  6. הגדר Slave ID (מספר שנקבע במכונה עצמה, בד"כ 1)
  7. הגדר טווח קוטרים — המערכת תפנה למכונה זו רק הזמנות בקוטר המתאים
  8. לחץ ➕ הוסף מכונה

✅
כיצד לדעת את Slave ID של המכונה? נכנס לתפריט ההגדרות של מכונת הכיפוף (דרך הלוח הקדמי) ומחפש "Station Address" או "Modbus ID". ברוב מכונות XINJE, ברירת המחדל היא 1.

בדיקת חיבור

לאחר שמירת ההגדרות, לחץ 📡 בדיקה בכרטיס המכונה. המערכת שולחת פקודת Modbus ומציגה "✅ מחובר" או הודעת שגיאה.

טווח קוטרים — למה זה חשוב?

כל מכונה מסוגלת לכופף קוטרים מסוימים בלבד. הגדרת הטווח מונעת שגיאות:

  • הזמנה עם ברזל Ø8–Ø12 תופנה למכונה קטנה (קוטר מינ' 6, מקס' 16)
  • הזמנה עם ברזל Ø32 תופנה למכונה כבדה (קוטר מינ' 16, מקס' 40)
  • אם אין מכונה מתאימה — מוצגת אזהרה

תחנות עבודה
מסך ליד המכונה לאופרטור

כל מכונה יכולה להיות מצוידת בטאבלט או מסך קטן המציג את ההזמנה הנוכחית. האופרטור רואה את הצורה, המידות, והכמות — ומדווח על השלמת כל שורת ייצור.

הגדרת תחנה

  1. ב-ניהול → תחנות עבודה, צור תחנה חדשה ושייך אותה למכונה
  2. הגדר PIN כניסה (4 ספרות)
  3. פתח /machine.html?machine=ID בטאבלט ליד המכונה
  4. הטאבלט יוצג עם מסך כניסה ב-PIN

שימוש יומיומי ליד המכונה

  1. הזן PIN להתחברות
  2. רשימת העבודות לאותה מכונה מוצגת אוטומטית
  3. לחץ ▶ התחל על הפריט הבא לייצור
  4. הפרמטרים נשלחים למכונה דרך Modbus
  5. לאחר ייצור — לחץ ✅ הושלם
  6. ספור הייצור מתעדכן בזמן אמת בדשבורד

חיבור מכונות כיפוף לרשת

חיבור דרך TCP/IP (מומלץ)

השיטה הנוחה ביותר — חיבור דרך הרשת הביתית של המפעל:

  1. חבר גיטוויי USR-N510 (או דומה) לפורט RS-485 של המכונה
  2. חבר את הגיטוויי לרשת ה-LAN של המפעל
  3. קבע לגיטוויי כתובת IP קבועה (static IP) — למשל 192.168.1.10
  4. ב-IronBend ← מכונות: הזן כתובת IP ופורט 502
  5. לחץ 📡 בדיקה

חיבור ישיר RTU (סיריאלי)

מתאים כשהשרת קרוב פיזית למכונה:

  1. חבר כבל RS-485 מהמכונה למחשב השרת (דרך מתאם USB→RS485)
  2. ב-Windows: פתח Device Manager → Ports ← ראה מה הCOM (למשל COM3)
  3. ב-Linux: הפורט יהיה בד"כ /dev/ttyUSB0
  4. ב-IronBend ← מכונות: בחר RTU, בחר פורט COM, הגדר Baud Rate
  5. Baud Rate חייב להתאים למה שמוגדר במכונה
⚠️
שגיאת "Timeout / No Response" בדוק: 1) כתובת IP נכונה? 2) Slave ID מתאים? 3) מכונה דלוקה? 4) בRTU — Baud Rate תואם? כבל תקין?

תשובות מהירות לשאלות שחוזרות

מכונה מוצגת כ"לא מחוברת"

  • ודא שהמכונה דלוקה והגיטוויי/מתאם מחוברים
  • בדוק שכתובת ה-IP בהגדרות תואמת לכתובת בגיטוויי
  • נסה 📡 בדיקה — ראה את הודעת השגיאה המדויקת
  • פינג לכתובת ה-IP מהשרת: ping 192.168.1.10

כפתור "שלח למכונה" לא עושה כלום

  • בדוק שהמכונה מוגדרת ומחוברת בניהול → מכונות
  • ודא שהקוטר של הפריט נמצא בטווח הקוטרים של המכונה
  • ראה לוג שגיאות ב-ניהול → לוג שינויים

הדפדפן מציג גרסה ישנה של הדף

לחץ Ctrl + Shift + R לרענון מלא (מנקה cache).

WhatsApp לא שולח הודעות

  • ודא ש-Token בתוקף (תפוגה: 24 שעות לטוקן זמני, ללא הגבלה לגרסה קבועה)
  • ודא שמספר הטלפון רשום ומאושר ב-Meta Business
  • בדוק בניהול → WhatsApp → לחץ 🔌 בדוק חיבור

איפוס סיסמת משתמש

עבור ל-ניהול → משתמשים → מצא את המשתמש → לחץ ✏️ ערוך → הזן סיסמה חדשה → שמור.
