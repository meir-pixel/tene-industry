1 ארכיטקטורה טכנית

⚙️ שרת (Backend)

  • Node.js 20 + Express 4
  • SQLite (better-sqlite3, WAL mode)
  • WebSocket (ws library)
  • Multer – העלאת קבצים/תמונות
  • node-cron – משימות מתוזמנות
  • CORS מופעל (פתוח לכולם)

🖥️ לקוח (Frontend)

  • HTML/CSS/JS ונילה – אין framework
  • Heebo (Google Fonts) – גופן ממשק
  • PWA – Service Worker + manifest.json
  • IndexedDB (offline-db.js) – שמירה offline
  • nav.js – ניווט משותף לכל הדפים
  • shape-editor.js v39 – עורך צורות

☁️ תשתית (Infra)

  • Render.com – Starter plan ($7/mo)
  • Persistent disk /data – 1GB (0.25$/GB)
  • DB_PATH=/data/ironbend.db
  • GitHub auto-deploy (push → deploy)
  • HTTPS מובנה ב-Render

🔗 חיבורים חיצוניים

  • Modbus RTU – מכונות כיפוף (דרך ממיר TCP)
  • Priority ERP – סנכרון הזמנות (Phase 2 כבוי עד חיבור אמיתי)
  • OpenAI API – ניתוח תמונות/טקסט
  • SMTP/IMAP – קבלת הזמנות במייל
  • Google Fonts – Libre Barcode 128 + Heebo

💾 מודל נתונים

  • הזמנה → 1..N משטחים → 1..N פריטים
  • פריט = צורה + קוטר + מידות + כמות
  • segments: JSON [{length_mm, angle_deg}]
  • משקל: kg/m לפי טבלת REBAR_WEIGHTS
  • 20+ טבלאות SQLite

📡 Real-time

  • WebSocket – wsBroadcast לכל הלקוחות
  • Events: new_order, order_status, machine_assign, item_complete
  • תחנת עבודה + מסך מכונות מתעדכנים בזמן אמת
  • Ping/Pong keepalive

shape-editor.js (v39)

יכולתפירוט
בחירת צורה14 presets מוכנות (ישר, L, U, Z, S, אוברל, אסדה, מלבן, ריבוע, 5-6-4 כיפופים, C, W). בחירה לפי כמות כיפופים (count picker) או מרשימה.
עריכה 2Dטבלת צלעות עם אורך וזווית כיפוף (1–179°). כפתורי 45°/90°/135°. תצוגה SVG עם labels ועיגולי זווית.
עריכה 3Dפנייה (azAngle -360..360°) והטיית Z (elAngle ±90°) לכל צלע. כפתורים: ישר/שמאל/ימין/הפוך. Orbit drag (camTheta/camPhi). ציר XYZ.
סנכרון 2D↔3DazAngles ↔ angles מסונכרנים בעריכה. טעינת צורה קיימת מחשבת azAngles מ-angles אוטומטית.
Overlap detectionצלעות מקבילות חופפות מוזזות בהיסט אלכסוני (OVLP_DX/DY ±4°) בשתי התצוגות.
שמירת צורהשמירה לטבלת shapes בDB. טעינת צורה שמורה.
Thumbnailsתמונות מוקטנות ב-count picker לכל preset.
Statsכיפופים, אורך בר, פרימטר כולל.

modbus.js

פונקציהתיאור
init(db)טעינת הגדרות מכונות מה-DB, ניסיון חיבור ראשוני.
getState(machineId)מצב נוכחי: connected/error/counter.
writeParams(id, {diameter, totalLengthMm, productionQty, angles})כתיבת פרמטרים ל-registers של מכונה. Protocol: Modbus RTU over TCP. Registers: defined per machine type.
Pollingקריאת counter כל N שניות → שידור WebSocket machine_counter.

offline-db.js

IndexedDB storepending_orders – שמירת הזמנות כשאין חיבור
Syncכשהחיבור חוזר → שליחה אוטומטית ל-/api/orders
PWAService Worker מאחסן בcache: HTML, JS, CSS, fonts

חישוב משקל

טבלת REBAR_WEIGHTS (kg/m): 6→0.222, 8→0.395, 10→0.617, 12→0.888, 14→1.21, 16→1.58, 18→2.00, 20→2.47, 22→2.98, 25→3.85, 28→4.83, 32→6.31, 36→7.99, 40→9.86

weight_per_unit = (total_length_mm / 1000) × kg/m

billing_weight = total_weight × (1 + waste_pct / 100) — ברירת מחדל waste_pct=3%

auto-assign מכונה

בהגשת הזמנה: קוטר ≤12mm → מכונה A; 14-20mm → B; >20mm → D
