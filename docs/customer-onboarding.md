## עיקרון

## שלב 0 — גילוי מקדים (לפני שנוגעים בטכני)
מלא טופס גילוי מול הלקוח. בלי זה — מפתיעים אותך באמצע ההקמה.

| שדה | למה צריך |
|------|----------|
| שם העסק + לוגו | מיתוג המערכת |
| תחום | מודול תעשייה (steel-rebar / עתידי) |

| חשבונות שלהם | WhatsApp? מייל? OpenAI? או שלך? |
| נתונים קיימים | רשימת לקוחות / מחירון לייבוא |
| דומיין מבוקש | `שם.tene-ind.com` |

🔨 **צריך:** טופס גילוי מסודר (Google Form / מסמך).

## שלב 1 — הקמת רישיון ושרת (אתה, ~15 דק')

1. ⚙️ **רישיון בפאנל** — `tene-license.onrender.com/admin` → רישיון חדש
   - שם, חבילה, מודולים, `max_users`, `billing_status` (free/trial/active) → קבל `LICENSE_KEY` ✅
2. ⚙️ **הרץ provision-customer** → מקבל בלוק env מוכן ✅
   ```
   node tools/provision-customer.js --name "מפעל כהן" --module steel-rebar --domain cohen.tene-ind.com
   ```
3. ⚙️ **שרת ב-Render** — Web Service מהריפו, rootDir=שורש, דיסק `/data` 1GB, הדבק env 🔨(תלוי בתיקון deploy)
4. ⚙️ **אתחול DB** — `ALLOW_EMPTY_DB_INIT=true` → deploy → החזר ל-`false` ✅
5. ⚙️ **דומיין מותאם** — `cohen.tene-ind.com` (CNAME ל-Render) + SSL אוטומטי 🔨(צריך דומיין)
6. ✅ **בדיקה** — health ירוק, גיבוי לילי רשום

## שלב 3 — גישה מאובטחת (לא לשלוח סיסמה בטקסט!)

12. ⚙️ **משתמש מנהל ללקוח** — שם משתמש + **סיסמה זמנית** 🔨
13. 🔨 **כפיית החלפת סיסמה בכניסה ראשונה** — חובה. סיסמה שנשלחה במייל אסור שתישאר. 🔨(צריך להיבנות)
14. ⚙️ **מייל ברוכים הבאים** (נוסח למטה) — קישור + פרטי כניסה

> ⚠️ **אבטחה:** לעולם לא לשלוח סיסמה קבועה במייל. או סיסמה חד-פעמית שמתחלפת בכניסה, או קישור הפעלה. זה ההבדל בין חובבני למקצועי.

    ```
    הזמנה → אישור → ייצור → השלמה → משלוח → חשבונית → גיבוי
    ```

**Go-live מאושר רק כשזרימה מלאה עברה.** לא לפני.
