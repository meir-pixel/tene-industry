# אפיון: היגיינת Frontend — קוד נקי לכל מסך

> בעלות: `public/theme.css`, `public/*.html`, `public/nav.js`. תחום GPT — אפיון בלבד.
> מקור: מאיר — "קוד נקי לכל מודול". הבאג של חלון החיפוש חשף את השורש.

---

## הבעיה — נמדדה בקוד, לא הנחה
רכיבים משותפים מוגדרים שוב ושוב בכל מסך, במקום פעם אחת:

| רכיב | בכמה מסכים מוגדר | צריך להיות |
|------|------------------|-----------|
| `.btn` | **16** | 1 (theme.css) |
| `.card` | **13** | 1 |
| `.tab-btn` | **7** | 1 |
| רכיב הניווט/חיפוש | **2** (theme.css + nav.js) | 1 |

**התוצאה:** כל שינוי צריך 16 מקומות; כשמתפספס — מסכים מתפצלים ויזואלית; וכשמגדירים פעמיים את אותו דבר — מתנגשים (בדיוק הבאג: `#ib-search-bar` ב-theme.css דרס את nav.js ופתח את החלון לבד).

זה ה-frontend המקביל לבעיה שכבר פתרנו ב-backend (שם API+DI נתנו מקור אחד לכל דבר).

---

## העיקרון — מקור אחד לכל סגנון
```
theme.css   = בסיס + רכיבים משותפים (.btn .card .tab-btn .table ...)  — פעם אחת
nav.js      = רכיב הניווט/חיפוש — בעל יחיד
<style> במסך = רק סגנון ייחודי למסך הזה. לא מגדיר מחדש רכיב משותף.
```

**החוק:** לכל סלקטור CSS יש **בית אחד**. אם `.btn` ב-theme.css — אסור להגדיר `.btn` בשום `<style>` של מסך.

---

## שלב 1 — לרכז רכיבים משותפים ל-theme.css
1. ודא ש-theme.css מגדיר במלואם: `.btn` (+וריאנטים), `.card`, `.tab-btn`/`.tabs`, `.table`/`.table-wrap`, `.field`/`label`, מודלים, badges.
2. מחק את ההגדרות הכפולות מה-`<style>` של כל מסך — **אחד אחד, עם אימות ויזואלי אחרי כל מסך.**

> ⚠️ **לא דה-דופ עיוור.** ניסיון להסיר את כל הכפילות בבת-אחת שבר את רוחב חלון החיפוש (רגרסיה). הכלל: מסך אחד בכל פעם → לבדוק חי → commit. אם משהו נשבר — מחזירים מיד.

---

## שלב 2 — בדיקת governance שתופסת כפילות
`test/css-hygiene.test.js`:
```javascript
// 1. אסוף את כל הסלקטורים מ-theme.css
// 2. עבור כל public/*.html — חלץ סלקטורים מתוך <style>
// 3. כשל אם סלקטור משותף (.btn .card .tab-btn ...) מוגדר גם ב-theme וגם במסך
// 4. אזהרה אם אותו סלקטור מוגדר ב-2+ מסכים (מועמד להעברה ל-theme)
// 5. אסור #ib-* / .ib-* מחוץ ל-nav.js (רכיב הניווט בעל יחיד)
```
ככה כפילות **חדשה** נחסמת אוטומטית — "קוד נקי" הופך לכלל אכיף, לא משמעת ידנית.

---

## שלב 3 — nav = בעל יחיד (בזהירות)
nav.js מחזיק את כל סגנון הניווט/חיפוש. theme.css לא מגדיר `#ib-*`/`.ib-*`.
**אבל** — ההסרה מ-theme.css חייבת אימות חי לכל מסך (הוכח שהיא יכולה לשבור). עד אז, לפחות לוודא שאין ערכים מתנגשים (כמו ה-display שכבר תוקן).

---

## למה לא לשכתב הכל עכשיו
שכתוב 28 מסכים = פרויקט ענק ומסוכן לפני לקוח ראשון. הגישה:
1. **לא לשכתב — לנקות תוך כדי.** כל מסך שנוגעים בו (למשל בסידור אזור החומר) — מנקים את הכפילויות שלו באותה הזדמנות.
2. **הבדיקה שומרת** שלא נוסיף כפילות חדשה.
3. עם הזמן — כל המסכים מתנקים בטבעיות.

---

## עקרונות
1. **מקור אחד לכל סלקטור.**
2. **theme.css = משותף · `<style>` = ייחודי למסך · nav.js = ניווט.**
3. **ניקוי מאומת חי, מסך-מסך — לא עיוור.**
4. **בדיקה אוכפת — כפילות חדשה נחסמת.**
