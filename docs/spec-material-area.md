# אפיון: ארגון מחדש של אזור החומר (רכש / קבלה / מלאי / מחסן)

> בעלות: `public/inventory.html`, `public/procurement.html`, `public/warehouse.html`, routes נלווים. תחום GPT — אפיון בלבד.
> מקור: מאיר — "לא מבין מה כל החלונות במחסן/מלאי/רכש".

---

## העיקרון — ארגון לפי זרימת החומר הפיזית
```
רכש → קבלת חומר → מלאי → ייצור → מחסן → משלוח
(קונה)  (נכנס)    (יש)   (צורך)  (ארוז)  (יוצא)
```
כל מסך = שלב אחד בזרימה. כיוון אחד בלבד לכל מסך — **לא מערבבים נכנס עם יוצא**.

---

## המצב היום — חפיפות
| מסך | לשוניות | בעיה |
|-----|---------|------|
| מלאי | סיכום, אצוות, **תעודות ספק**, **ספקים**, פחת | ספקים כפול, קבלה כפולה |
| רכש | הזמנות רכש, **ספקים**, מחירי ברזל | ספקים כפול |
| מחסן | מחסן, טעינה, **קבלת חומר גלם** | מערבב נכנס (קבלה) עם יוצא (טעינה) |

3 כפילויות: **ספקים** ×2, **קבלת חומר** ×3, וכיוונים מעורבבים במחסן.

---

## המבנה הנקי המוצע

### 1. רכש (procurement.html) — "מה אני קונה מהספקים"
```
לשוניות:  הזמנות רכש  |  ספקים  |  מחירי קנייה (ברזל)
```
- **בית יחיד של ספקים** — כאן ורק כאן.
- **מחירי קנייה** = מה שאני משלם לספק (שונה ממחירון המכירה ללקוח).
- הסר "ספקים" מהמלאי.

### 2. מלאי (inventory.html) — "מה יש לי עכשיו + קבלת חומר"
```
לשוניות:  סיכום מלאי  |  אצוות / מנות  |  קבלת חומר (OCR)  |  פחת וגרוטאות
```
- **קבלת חומר = בית יחיד.** חומר נכנס מהספק → צילום תעודה → OCR → אישור → נכנס למלאי.
- הסר "ספקים" (עבר לרכש). הסר "קבלת חומר" מהמחסן.
- "תעודות ספק לאישור" + "קבלת חומר" מתאחדים ללשונית אחת.

### 3. מחסן (warehouse.html) — "מה יוצא ללקוח" (יוצא בלבד!)
```
לשוניות:  חבילות מוכנות  |  טעינה ומשלוח
```
- **רק כיוון יוצא** — חבילות ארוזות + טעינה לרכב.
- **הסר "קבלת חומר גלם" מכאן לגמרי** — זה נכנס, שייך למלאי.

## באג מיידי לתקן
**חלון חיפוש קופץ אוטומטית** בכניסה למסך המלאי ("חיפוש הזמנה, לקוח, חבילה..."). הוא לא אמור להיפתח לבד — רק בלחיצה על אייקון חיפוש. צריך להסיר את הפתיחה האוטומטית.

---

## OCR קבלת חומר — מצב
ה-OCR לקבלת חומר **קיים** (`/inventory/receipt-reviews/analyze`, `/inventory/scan-label`), אבל:
- כבוי כברירת מחדל (`INTAKE_AI_ENABLED=false`) — צריך הפעלה + מפתח OpenAI ב-Settings.
- מסך ההשוואה מקור↔זיהוי טעון שיפור (ראה `spec-intake-ux.md` בעיה B — אותו פתרון: צד-ליד-צד).

## סיכום שינויים
| פעולה | פירוט |
|-------|-------|
| ספקים → רכש בלבד | הסר מהמלאי |
| קבלת חומר → מלאי בלבד | הסר מהמחסן, אחד את הכפילות במלאי |
| מחסן = יוצא בלבד | הסר "קבלת חומר גלם" |
| מחירי קנייה → רכש | נפרד ממחירון מכירה |
| תקן באג חלון חיפוש קופץ | במלאי |

## עקרונות
1. **כיוון אחד למסך** — נכנס (קבלה/מלאי) נפרד מיוצא (מחסן).
2. **בית יחיד לכל ישות** — ספק במקום אחד, קבלה במקום אחד.
3. **שם = פעולה** — לשונית עושה דבר אחד ברור.

- [ ] מחירי קנייה ברכש, נפרד ממכירה
- [ ] באג חלון חיפוש קופץ — תוקן
