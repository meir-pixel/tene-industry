# אפיון: הפרדת ניהול ספק / לקוח + מודל גיבוי בענן

> סטטוס: אפיון מוכן. נוגע בבעלות משותפת — שרת הרישיונות (Claude) + admin של האפליקציה (GPT).

## הבעיה
ניהול המערכת רץ היום בצד הלקוח. חלקו אסור שיהיה בידי הלקוח (מודולים, רישיון, מפתחות ספק). לקוח טכני יכול לעקוף רישוי.

## ההחלטה
- **השרת של כל לקוח רץ בענן** (לא מקומי).
- **ניהול ספק** יושב **בשרת הרישיונות של Tene** — הלקוח לא נוגע.
- **ניהול לקוח** נשאר בשרת הלקוח — רק מה ששלו.
- **אין גיבוי מקומי** — רק לשרת הגיבוי של Tene (מקומי = חשוף/מסוכן).

## הפרדת אחריות

### ניהול ספק — שרת הרישיונות (Tene בלבד)
| מה | היכן |
|----|------|
| אילו מודולים פתוחים לכל לקוח | פאנל הרישיונות |
| מכסת משתמשים | פאנל הרישיונות |
| תוקף/ביטול רישיון | פאנל הרישיונות |
| מפתחות API של Tene (אם יש) | env של שרת הרישיונות |
| גיבויים של כל הלקוחות | שרת הגיבוי |

### ניהול לקוח — שרת הלקוח (מנהל הלקוח)
| מה | היכן |
|----|------|
| משתמשים של הלקוח | admin אצלו |
| הגדרות מותרות (waste%, עלויות, התראות) | settings אצלו (`customer_permission='edit'`) |
| WhatsApp/מייל של הלקוח | settings אצלו |
| לקוחות, הזמנות, מלאי | האפליקציה שלו |

## מה צריך לוודא בצד הלקוח (admin.html / routes/admin.js)
1. **להסיר/להסתיר כל כפתור ספק** — אין ב-admin של הלקוח: בחירת מודולים, מכסת משתמשים, מפתח רישיון, החלפת מודול תעשייה.
2. **`vendor_only` מוסתר** — settings עם `vendor_only=1` לא מוצגים בכלל ללקוח (כבר קיים — לוודא נאכף ב-UI).
3. **`license_modules` לקריאה בלבד** — הלקוח רואה אילו מודולים יש לו, לא יכול לערוך. מקור האמת = שרת הרישיונות.

## האמת הכנה על הגנה
גם בענן, אם הלקוח מקבל גישת ניהול לשרת שלו — אפשר לעקוף. אבל:
- בענן Tene, הלקוח **לא מקבל גישת מערכת** לשרת — רק לאפליקציה דרך הדפדפן.
- המנוף האמיתי: הרישיון (נבדק מול שרת Tene) + הגיבוי (אצל Tene) + חוזה.

## מודל גיבוי — בלי מקומי

```
שרת לקוח (ענן) ──גיבוי לילי מוצפן──► שרת הגיבוי (Tene) ──► אחסון (דיסק/B2)
```

**שלוש שכבות הגנה, כולן אצל Tene:**
1. גיבוי לילי מוצפן AES-256 → שרת הגיבוי
2. snapshot יומי של דיסק Render (אוטומטי)
3. שחזור בדוק — `tools/restore-backup.js`

**אין עותק אצל הלקוח** — הוא לא יכול לדלוף/להשחית/לזייף.

**אינטרנט נופל במפעל?** השרת בענן ממילא דורש חיבור — הפתרון: ראוטר סלולרי גיבוי במפעל (~80₪/חודש), לא גיבוי מקומי.
